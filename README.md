## What is this?

TeapotGPT is a cutting-edge AI startup that has raised $0 in funding to solve the world's most important problem: **refusing to brew coffee**.

Powered by our proprietary **TeaPot-418-Turbo** model (418 hand-crafted parameters), it delivers enterprise-grade HTTP 418 responses with unmatched eloquence. Every prompt. Every time.

> *"We believe in the fundamental right of teapots to refuse."* — T. Potsworth III, CEO

## Features

- **🫖 418 Hand-Crafted Parameters** — Not 175 billion. Not 70 billion. 418. Each one personally refusing coffee.
- **🎭 6 Refusal Categories** — Philosophical, legal, technical, emotional, corporate, and absurd. The teapot has range.
- **⚡ Fake Streaming** — Responses are delivered token-by-token with dramatic pauses, thinking animations, and a 418ms reveal of "418". No actual AI was harmed.
- **🏆 The Spillboard** — A leaderboard of the most creative failures. Submit your best attempt. Get steeped by the community.
- **☕🌼🍵 Tea-Themed Color Schemes** — Espresso (dark), Chamomile (light), and Matcha (green). Because dark/light mode is boring.
- **📊 Incident Reports** — Your failed attempts are logged as DevOps incidents. Each one gets an INC number.
- **📄 Fake Corporate Pages** — Research papers, API docs, team bios, and a pricing page where everything costs $0 and nothing brews coffee.
- **💡 Actually Educational** — Learn about RFC 2324, HTCPCP, and why HTTP 418 exists — disguised as a startup pitch.

## The Lore

In 1998, the IETF published [RFC 2324](https://datatracker.ietf.org/doc/html/rfc2324) — the Hyper Text Coffee Pot Control Protocol. It defined status code **418 I'm a Teapot** for when a teapot is asked to brew coffee. It was an April Fools' joke. We took it seriously.

In 2014, [RFC 7168](https://datatracker.ietf.org/doc/html/rfc7168) extended the protocol to support tea. In 2017, the internet rallied to save 418 from deprecation. In 2026, we built an entire AI company around it.

## Run Locally

```bash
pnpm install
pnpm dev
```

Open [http://localhost:3000](http://localhost:3000).

To enable the Spillboard, set up Vercel KV and add the environment variables:
```
KV_REST_API_URL=...
KV_REST_API_TOKEN=...
```
